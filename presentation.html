Problem Statement

The Mental Health Accessibility Crisis

😰

970 Million Affected

1 in 8 people worldwide live with a mental health disorder, yet the majority never receive any form of professional care.

💬

Text-Only Chatbots Fail

Existing AI chatbots only analyze text input. They miss the non-verbal emotional signals visible in facial expressions — the richest source of human emotion.

🌐

24/7 Support Gap

Human therapists are limited by time, cost, and availability. Existing digital tools feel cold, robotic, and lack continuity of empathy.

"We need a system that doesn't just read what users say — but one that sees and truly understands how they feel."

Objective

What We Set Out to Build

To design and develop Mindful Companion — a state-of-the-art AI wellness platform that combines Computer Vision (facial emotion recognition) with Generative AI (LLM-powered chat), creating an always-available, emotionally intelligent support system that adapts its tone and advice in real-time based on the user's detected emotional state.

🎯 Primary Goals

Requirements

Functional Requirements

🖥️ FR-01 · Chat Interface

The system shall provide a real-time chat interface where users type messages and receive AI-generated empathetic responses streamed word-by-word via SSE.

🎭 FR-02 · Facial Emotion Detection

The system shall use the webcam feed to detect and classify user facial emotions (Happy, Sad, Anxious, Fearful, Neutral, Angry, Disgusted) in real-time using DeepFace.

💉 FR-03 · Emotion Injection

The detected facial emotion shall be injected into the LLM's system prompt context, enabling tone-adaptive responses (e.g., gentler language when sadness is detected).

🚨 FR-04 · Crisis Detection

The system shall scan every message for crisis keywords (self-harm, suicide) and immediately override any LLM response with emergency contact information.

🔒 FR-05 · Scope Guardrail

The system shall detect non-wellness queries (coding, ML, legal) and redirect the user to emotional support instead of attempting to answer technical questions.

📊 FR-06 · Wellness Toolkit

The system shall provide a Mood Tracker with charts, a private Digital Journal, and a Resource Hub with mental health articles and emergency contacts.

AI Models

Models Used

☁️
Tier 1 · Primary

Google Gemini 1.5 Flash

Cloud LLM via LangChain. Warm, empathetic, contextual wellness responses. Supports streaming output. Falls back to GPT-4o.

Empathy quality: 95% · Real-time streaming

🤖
Tier 2 · ML Fallback

Scikit-Learn Classifier

Trained on the Mental Health Chatbot Dataset. Uses TF-IDF vectorization with a supervised classifier saved as mental_health_model.pkl.

Classification accuracy: ~78% · Offline

📘
Tier 3 · Heuristic

Rule-Based Heuristic Engine

Pattern-matching over MentalHealthChatbotDataset.json. Keyword overlap scoring (Jaccard-style). Always guaranteed to respond.

Relevance: ~60% · 100% uptime

🎭 DeepFace — Facial Emotion Recognition Model

Pre-trained deep neural network (VGG-Face / FaceNet backbone) wrapped in the DeepFace library. Accepts base64-encoded webcam frames via Flask and returns a dominant emotion label among 7 classes: Happy · Sad · Angry · Fear · Disgust · Surprise · Neutral. Combined with OpenCV for frame pre-processing.

System Workflow

Request Processing Flow

1

User Interaction

User types a message in the React chat UI. Simultaneously, FaceScanner captures a webcam frame and sends it to /analyze_emotion — receiving an emotion label (e.g., "Sad").

2

Scope Guardrail Check

Backend scans the message for tech/coding keywords. If detected → returns a wellness redirection without involving the LLM. (Keeps the AI focused on mental health.)

3

🚨 Crisis Safety Override

Hard-coded keyword scanner checks for self-harm / aggression phrases. If triggered → immediately returns crisis hotline information (112, 988). No LLM involvement for safety.

4

Emotion + Message → Intelligence Orchestrator

Emotion label is merged into the LLM system prompt. Orchestrator attempts: Gemini 1.5 Flash → GPT-4o (GitHub Models) → Scikit-Learn ML → Heuristic JSON Engine.

5

SSE Streaming Response

Response is streamed back to the React frontend via Server-Sent Events (text/event-stream). Frontend renders tokens word-by-word for a natural conversational feel. Session memory (6-turn deque) is updated.

Key Modules

Core Modules & Components

🏛️ ModelManager (Backend)

Central orchestrator class in app.py. Manages initialization and invocation of all three AI tiers. Holds session memory (6-turn deque per session ID), emotion emoji mapping, crisis keyword list, and humanization logic.

📘 HeuristicModel (Backend)

Loads MentalHealthChatbotDataset.json and performs Jaccard-style overlap scoring between user input and intent patterns. Prevents repeat responses via non-repeating response picker.

🎭 FaceScanner.tsx (Frontend)

React component that accesses the webcam using the MediaDevices API, captures frames at intervals, base64-encodes them, and POSTs to /analyze_emotion. Displays live emotion overlay on the video feed.

💬 Chat.tsx (Frontend)

Full-featured chat page that listens to the SSE stream from /chat_stream. Renders emotion badges alongside each message bubble and appends streamed tokens in real-time to the UI.

📊 MoodTracker.tsx (Frontend)

Interactive page displaying time-series emotional trend charts. Visualizes the user's mood history to help identify patterns and emotional progress over time.

🛡️ Safety & Guardrail Module

Two hard-coded guardrail functions: _safety_override() (crisis detection) and _non_mental_health_redirect() (scope guard). Both run before any AI model is invoked.

Challenges & Future Scope

What We Overcame & What's Next

⚔️ Challenges Faced

⚠️

DeepFace Latency

Frame inference took 2–4s initially. Solved by reducing resolution and processing frames asynchronously, only on user submit.

⚠️

ML Misclassification

"My name is John" was classified as Sadness. Fixed via name-intro guardrail pattern matching.

⚠️

API Rate Limits

Gemini quota exhaustion during testing. Solved with exponential backoff + multi-provider fallback chain.

⚠️

CORS & Streaming

SSE requires special HTTP headers. Added X-Accel-Buffering: no and correct CORS headers for all OPTIONS preflight requests.
